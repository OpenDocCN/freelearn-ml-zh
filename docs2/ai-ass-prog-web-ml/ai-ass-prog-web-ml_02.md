# 你所在的位置

作为数据科学和/或全栈网络开发的读者和实践者，您了解您的技艺。了解您的技艺意味着您知道解决问题的关键工具和技术。在这个阶段，您正在查看一个人工智能助手，并意识到它是由自然语言，即所谓的提示所控制的。您可能没有意识到，这不仅仅是编写一个提示并得到一个答案那么简单。人工智能助手是在大量文本语料库上训练的，因此它在可以生成文本的内容以及如何响应提示方面非常灵活。正因为这种灵活性，理解如何编写有效且高效的提示就变得非常重要。

# 如何高效地提示的指南

提示是人工智能工具的输入。根据你试图实现的目标，你需要调整你的提示以适应你正在解决的问题场景。因此，你如何“提示”很重要。例如，如果你的提示太模糊，你将不会得到你需要的。或者，假设你试图使用提示来生成公司口号；你不希望为生成应用程序代码使用相同的提示。相反，在数据科学这样的学科中，按照一定的顺序执行任务很重要，你的提示应该反映你想要完成的工作，以及如果需要，完成这些工作的步骤。

你需要成功的是一种方法，一种策略，你可以在一般情况下使用它来与人工智能助手高效地合作。此外，这样的策略应该足够具体，以便为所选问题领域提供“最佳实践”。正如本章前面提到的，我们已经为该领域的全栈 Web 开发和数据科学开发了一种特定的提示策略。

在高层次上，我们建议使用人工智能助手进行一般问题解决的指南；我们相信它适用于你将提示应用于任何问题领域的情况：

1.  **将问题分解**，以便完全理解。在这个指南中，可能会有以下步骤：

    +   **理解问题**：对于任何问题，理解问题是什么以及它不是什么都很重要。例如，我们是构建一个用于预测销售的机器学习模型，还是一个用于跟踪库存的网页？这两个问题是不同的，需要不同的方法。

    +   **识别各个部分**：一个问题通常很复杂，由许多需要解决的组成部分组成。例如，如果我们正在构建一个用于预测销售的机器学习模型，我们需要识别数据、模型、训练和评估。这些部分的每一部分都可以进一步分解成更小的部分。一旦你找到了适合你问题的正确细节级别，你就可以通过编写提示来开始解决问题。

    +   **将问题分解成更小的部分**：如果需要，将你的问题分解成更小、更易于管理的部分。

    +   **识别和理解数据**：特别是在机器学习中，确定一个可以工作的数据集、它包含的内容以及它的结构至关重要。在 Web 开发中，数据也扮演着核心角色，但目标通常是确保你能够以对用户有用的方式读取、写入和展示数据。

1.  **在适当的级别生成提示**。一旦你完全理解了问题，你应该有一个任务列表，并且对于每个任务，你应该能够编写和运行一个解决该任务的提示。

1.  **验证解决方案**。就像没有 AI 助手一样，验证在构建系统或应用中起着至关重要的作用。传统上，这意味着编写测试，测试各种组件的组合，并让用户尝试各种部分。使用提示没有不同。LLMs 的一个副作用是它们可以生成与问题不相关的文本，或者以非最佳方式解决问题。因为您依赖于最终为您生成代码的提示，所以验证解决方案以确保其正确性和相关性变得尤为重要。

在接下来的章节中，我们将描述为每个选定的问题域的全栈网络开发和数据科学制定详细策略。

## 提示技术

有一些技巧在指导 AI 助手时特别有用，以确保您充分利用它。一般建议是向 AI 助手提供足够的背景信息，但具体意味着什么可以通过以下建议的模式来具体化：

+   任务-行动-指南提示模式

+   **人物-指令-内容提示模式**

+   探索性提示

### 任务-行动-指南提示模式（TAG）

为了解释这个模式是如何工作的，让我们分解各个部分，并展示一个带有示例响应的示例提示：

**意图**：指导 AI 助手解决什么任务，采取什么行动，以及遵守什么指南。

+   **任务**：明确定义您希望 AI 完成的任务

+   **行动**：指定 AI 应采取的步骤或行动以完成任务

+   **指南**：提供 AI 在执行任务时应遵循的任何规则、约束或指南

**示例**：

**[提示]**

**任务**：生成关于最新气候变化新闻文章的摘要。

**行动**：

1.  搜索有关气候变化的最新新闻文章。

1.  选择相关性最高、可信度最高的文章。

1.  用 3-4 句话总结文章的关键点。

**指南**：

1.  确保摘要简洁准确。

1.  避免使用技术术语；保持语言简单易懂。

1.  在摘要末尾引用文章来源。

**[提示结束]**

**[提示响应]**

**摘要**：BBC News 最近的一篇文章突出了过去十年全球温度的惊人上升。科学家们警告，如果当前趋势持续下去，我们可能会看到前所未有的气候事件。文章强调，国际社会迫切需要合作来应对气候变化。[来源：BBC News]

这种模式有助于确保 AI 助手理解任务，知道采取的步骤，并遵守任何具体指南，从而产生更准确、更有用的输出。

**[响应结束]**

下面是一个关于网络开发的提示示例：

**[提示]**

**任务**：使用 Node.js 和 Express 为简单的博客应用程序开发 RESTful API。

**行动**：

1.  设置一个新的 Node.js 项目。

1.  创建一个 Express 服务器。

1.  定义博客文章的 CRUD 操作路由。

1.  实现 JSON 解析和错误处理的中间件。

1.  使用 Mongoose 连接到 MongoDB 数据库。

1.  为 API 端点编写测试。

**指南**：

1.  确保项目设置清晰易懂。

1.  提供简洁准确的代码示例。

1.  避免使用过于复杂的术语；保持语言简单易懂。

1.  在代码中包含注释以解释每一步。

**[提示结束]**

### 角色指令-上下文提示模式（PIC）

让我们以相同的方式描述这个模式，先定义它，然后是一个示例提示和一个示例响应：

**意图**：指导 AI 助手扮演的角色，执行哪些指令，并提供足够的上下文以生成正确的响应。

+   **角色**：定义 AI 在响应时应体现的角色或性格特征

+   **指令**：指定 AI 应执行的任务或动作

+   **上下文**：提供相关的背景信息或上下文，以帮助 AI 理解情况并生成适当的响应

示例：

**[提示]**

**角色**：扮演一个友好且知识渊博的旅行指南。

指令：

1.  推荐巴黎三个必游的景点。

1.  为每个景点提供简要描述。

1.  建议参观每个景点的最佳时间。

上下文：用户第一次计划去巴黎旅行，对历史遗迹和当地文化都感兴趣。

**[提示结束]**

**[提示响应]**

作为您友好且知识渊博的旅行指南，我很乐意推荐巴黎三个必游的景点：

1. **埃菲尔铁塔**：这个标志性的地标提供了令人叹为观止的城市景观。最好在清晨或傍晚时分参观，以避开人群，欣赏壮丽的日出或日落。

享受您的巴黎之旅！

**[响应结束]**

这种模式确保 AI 助手采用正确的角色，遵循特定的指令，并使用提供的上下文提供相关且有帮助的响应。

### 学习-即兴发挥-反馈-评估提示模式（LIFE）

这种模式，就像 TAG 和 PIC 一样，有助于界定问题并提供一个良好的解决方案起点，你可以进一步细化。

+   **学习**：强调通过各种分析技术理解数据的重要性，从基本统计到复杂的相关性和时间分析。

+   **即兴发挥**：根据初步发现调整分析。例如，如果某些产品类别显示出不寻常的趋势，就深入分析这些领域。

+   **反馈**：

    +   分享代码和模型输出以供反馈，以确保有效的学习和理解。

    +   将建议和批评融入以完善模型和方法。

    +   提供错误以排查和解决这些问题

+   **评估**：执行 ChatGPT 提供的代码以确保准确性和有效性。

+   **学习**：理解有效地显示搜索结果的重要性。分析用户交互数据以识别关键指标，如点击率、平均在结果页面上的时间以及用户满意度评分。使用从基本统计到复杂相关性及时间分析的各种分析技术，深入了解用户行为和偏好。

+   **评估:** 执行 ChatGPT 提供的代码以确保其准确性和有效性。通过将性能与实际数据和用户反馈进行比较来验证搜索结果页面。根据需要调整以改进准确性和用户满意度。

### 选择哪种模式？

一般答案是取决于具体情况选择哪种模式。选择 TAG 和 PIC 的原因是你有一个从开始到结束都希望解决的明确问题，并且你希望 AI 助手来做“重活”。选择探索模式的原因是当你正在处理现有的代码库，或者你想要尝试各种方法和步骤。我们的普遍建议是尝试所有三种模式，看看哪种最适合你的方法和问题。

# Web 开发的提示策略

让我们应用前一部分中提出的不同指南，通过使用这些指南，我们将逐步工作，逐步制定出提示策略。

## 将问题分解：“库存管理系统”

让我们用一个真实例子，“Web 开发中的库存管理”，来看看我们是否能够理解一般问题。要“管理”库存，你需要读取和写入数据。很可能会在这个系统/应用程序中有不同的角色，从管理员到普通用户。你可能还需要考虑这个系统如何与其他系统结合，例如，如果你需要将其与其他系统集成，那么它由哪些部分组成以及如何。

领域看起来相当直接，让我们继续了解它由哪些部分组成。

在高层次上，我们理解系统应该做什么。但为了解决问题，我们需要将其分解成更小的部分，在 Web 开发中通常包括以下组件：

+   **前端**：前端是用户与之交互的系统部分。前端负责向用户展示数据并接收用户输入。

+   **后端**：这个系统部分与前端通信。后端负责读取和写入数据库中的数据。在更复杂的系统中，可能还有不同的前端和不同的应用程序与后端进行更复杂的通信。

+   **数据库**：数据库是存储数据的系统部分。它是一个数据存储，例如，一个关系型数据库，如 MySQL 或 PostgreSQL。数据库负责以高效且易于读写的方式存储数据。

+   **报告**：通常有一个报告部分来展示洞察。它从数据存储中获取数据，可能需要转换数据以便从报告的角度展示。

## 前端功能的进一步分解

有这样的概览是有用的，但通常不足以让我们开始编写提示。我们需要进一步分解，通常是通过功能。在这个阶段，前端功能的进一步分解可能看起来像以下列表：

+   **登录**：用户需要能够登录到系统中。

+   **登出**：用户需要能够从系统中登出。

+   **查看库存**：用户需要能够查看库存。

+   **添加库存**：用户需要能够添加库存。

+   **移除库存**：用户需要能够移除库存。

+   **更新库存**：用户需要能够更新库存。

## 为每个功能生成提示

到这一点，它足够具体，我们可以开始编写提示。让我们以第一个功能，登录，来看看我们如何开始思考如何制作提示。

你可以从一个提示开始，使用像 ChatGPT 或 GitHub Copilot（更多内容将在下一章中介绍）这样的工具，如下所示：

**[提示开始]**

为用户创建一个登录页面以便登录系统。

**[提示结束]**

虽然这可能可行，但你遗漏了很多上下文，比如这里到底需要什么，什么是不需要的，你使用的技术是什么，用户体验如何等等。

让我们尝试通过添加更多上下文来改进提示：

**[提示开始]**

创建一个包含用户名和密码字段的登录页面。它应该有一个创建用户的链接和一个登录按钮，垂直和水平居中，并且在手机和平板电脑上运行良好。它应该使用 React 编写并使用 Material UI 库。

**[提示结束]**

## 确定一些 Web 开发的基本原则，一个“提示策略”

如你所见，我们将问题分解成更小、更“可管理”的部分，并建议了一些针对特定功能的提示。那么，现在我们了解了更多关于我们的示例“库存管理”，我们具体建议了什么“提示策略”？一方面，我们意识到我们的策略将是上下文相关的。因为我们处于 Web 开发领域，我们需要在该领域使用关键词，以及适合特定区域的库和架构。以下是我们建议你使用的指南：

+   **提供上下文 - 字段**：登录屏幕可以像用户名和密码字段那样简单。然而，大多数屏幕有更多的字段，例如密码确认字段、重置密码的链接、创建新用户的字段等等。根据你的需求，你可能需要非常详细。

+   **指定如何 - 设计和技术选择**：登录屏幕可以以多种方式设计。如今，优化不同设备如平板电脑、手机、大屏幕等是非常常见的。在技术选择方面，Web 开发有很多选择，从 React、Vue 和 Angular 这样的框架到纯 HTML 和 CSS。根据项目的需求进行指定。

+   **迭代**：不同的工具对相同的提示反应不同。在这本书中，我们将向你展示如何使用 GitHub Copilot 和 ChatGPT 等不同的工具。每个工具都有其自身的优势和劣势，可能会提供不同的结果。尝试通过添加分隔符如逗号和冒号来迭代提示，并尝试重新措辞提示。

+   **具有情境意识**：当你使用像 ChatGPT 和 GitHub Copilot 这样的工具时，你是带着预先存在的情境使用的。对于 ChatGPT 来说，这意味着你正在进行一个持续的提示和响应对话，而对于 GitHub Copilot 来说，这意味着它不仅看到了你在打开的文件中写的内容，如果你允许的话，还包括你的整个工作空间。对你的提示的响应会考虑这个情境，并决定生成什么。了解这个情境很重要，如果你没有得到你想要的响应，尝试改变情境，在 ChatGPT 中开始新的对话，在 GitHub Copilot 中关闭打开的文件，开始在一个空白文件中写作等。

# 数据科学中的提示策略

让我们对数据科学进行类似的思维实验，就像我们对网络开发所做的那样。我们将使用提供的指南“问题分解”和“生成提示”，就像在 Web 开发部分一样，我们将对领域进行一些一般性的结论，并将这些作为数据科学的提示策略。

## 问题分解：预测销售

假设我们正在构建一个机器学习模型来预测销售。在较高层次上，我们理解系统应该做什么。但是，为了解决这个问题，我们需要将其分解成更小的部分，在数据科学中通常包括以下组件：

+   **数据**：数据是系统中存储信息的一部分。数据可以来自许多地方，如数据库、网络端点、静态文件等。

+   **模型**：模型负责从数据中学习并产生尽可能准确的预测。为了进行预测，你需要一个输入，它会产生一个或多个作为预测的输出。

+   **训练**：训练是系统中训练模型的部分。在这里，你通常有一部分数据作为训练数据，另一部分作为样本数据。

+   **评估**：为了确保你的模型按预期工作，你需要对其进行评估。评估意味着使用数据和模型产生一个分数，该分数表示模型的表现有多好。

+   **可视化**：可视化是你可以通过图表获得对业务有价值的见解的部分。这部分非常重要，因为它是业务中最可见的部分。

## 将数据科学分解为特征/步骤

在这个阶段，你处于一个过于高的层次，无法开始编写提示。我们可以通过查看每个步骤来进一步分解它：

+   **数据**：数据部分有许多步骤，包括收集数据、清洗数据和转换数据。以下是你可以如何分解它：

    1.  **收集数据**：数据需要从某个地方收集。它可能来自数据库、网络端点、静态文件等。

    1.  **清洗数据**：数据需要被清洗。清洗意味着移除不相关的数据、删除重复项等。

    1.  **数据转换**：数据需要被转换。转换意味着将数据改变成对模型有用的格式。

+   **训练**：就像数据部分一样，训练部分有很多步骤。以下是你可以如何分解它：

    1.  **分割数据**：数据需要被分割成训练数据和样本数据。训练数据用于训练模型，样本数据用于评估模型。

    1.  **训练模型**：模型需要被训练。训练意味着从训练数据中学习。

+   **评估**：评估部分通常是单一步骤，但可以进一步分解。

## 为每个步骤生成提示

注意我们的数据科学分解看起来与网络开发略有不同。我们不是识别像**添加库存**这样的功能，而是有一个像**收集数据**这样的功能。

然而，我们已经到达了编写提示的正确水平，所以让我们以**收集数据**功能为例：

**[提示]**

使用 Pandas 库从`data.xls`收集数据并将其读入 DataFrame。

**[提示结束]**

前面的提示既通用又具体。在通用方面，它告诉你要“收集数据”，但在具体方面，它指定了要使用的特定库以及数据结构（DataFrame）。完全有可能一个更简单的提示就可以完成前面的步骤，如下所示：

**[提示]**

从 data.xls 中收集数据。

**[提示结束]**

这取决于你是否使用像 ChatGPT 或 GitHub Copilot 这样的工具。

## 确定数据科学的一些基本原则，“数据科学提示策略”

这里，我们确定了与网络开发示例中类似的一些原则：

+   **提供上下文 – 文件名**：CSV 文件可以有任意名称。指定文件名很重要。

+   **指定如何 – 库**：有许多方法可以加载 CSV 文件，尽管 Pandas 库是一个常见的选择，但指定它很重要。还有其他库可以工作，你可能需要一个针对 Java、C#和 Rust 等语言的解决方案，在这些语言中库的名称不同。

+   **迭代**：迭代提示是有价值的，重新措辞它，并添加分隔符，如逗号、冒号等。

+   **注意上下文**：在这里，上下文也很重要；如果你在 Notebook 中工作，之前的单元格将可供 GitHub Copilot 使用，之前的对话将可供 ChatGPT 使用，等等。

如前所述的指导所示，策略在网页开发中也非常相似。这里我们也列出了“提供”、“指定如何”、“迭代”和“注意上下文”。主要区别在于细节。然而，在数据科学中还有一个有效的替代策略，那就是长提示。尽管我们已经将数据科学问题分解为功能，但我们不需要为每个功能编写提示。另一种解决方法可能是在一个大的提示中表达你想要执行的所有内容。因此，这样的提示可能看起来像这样：

**[提示]**

你想要预测`data.xsl`文件上的销售额。使用 Python 和 Pandas 库。以下是你应该执行的步骤：

+   收集数据

+   清洁数据

+   转换数据

+   分割数据

+   训练模型

+   评估

**[提示结束]**

你将在未来的章节中看到数据科学和机器学习中的例子，其中既使用了较短的提示，也使用了较长的提示。你决定使用哪种方法。

# 验证解决方案

这种策略最重要的部分是验证正确性，以及 AI 助手创建的文本和代码是否正确。我们可以采取两种一般方法来验证我们的结果：

+   **通过提示进行验证**：第一种方法是使用提示来验证结果。这意味着编写提示来质疑特定结果的输出。这可以在验证过程的开始时采用的一种好策略。你所寻找的是 AI 助手在响应中不一致的情况。

+   **经典验证方法**：第二种方法是使用经典验证技术。这些技术取决于问题领域。然而，从高层次来看，这归结为测试代码、比较输出，并依靠你自己的知识和你同伴的知识来验证结果。

AI 工具实际上并不知道它在做什么。提供的响应可能是依赖于其训练语料库的响应。在所有时候，你应该意识到这一点，并依靠你的专业知识来验证结果。

在接下来的小节中，让我们探讨各种手动和经典验证的方法。

## 通过提示进行验证

你可以使用提示来产生结果，这些结果可以帮助你更接近解决问题，也可以用来验证结果。让我们以我们正在构建之前提到的登录屏幕为例。我们已经编写了一个如下所示的提示：

**[提示]**

创建一个登录页面，包含用户名和密码字段；它应该有一个创建用户和登录按钮的链接。它应该垂直和水平居中，并在手机和平板上运行良好。它应该使用 React 编写并使用 Material UI 库。

**[提示结束]**

为了验证这个结果，我们可以写一个如下所示的提示：

**[提示]**

给定以下代码，它做什么？

```py
.login {
  <!-- should include CSS to center horizontally and vertically and be responsive -->
  @media (min-width: 768px) {
    <!-- should include CSS to center horizontally and vertically and be responsive -->
  }
}
<div class="login">
  <TextField id="username" label="Username" />
  <TextField id="password" label="Password" />
  <Button variant="contained" color="primary">
    Login
  </Button>
</div> 
```

**[提示结束]**

一个应用程序可以查看提供的代码，并意识到你不需要提示来推断它做什么，实际上它缺少 CSS 代码来使其响应。然而，这里的重点是，通过编写提示，你可以让 AI 助手通过提问告诉你它认为代码做什么。

使用提示这种方式，对输出提出查询，是验证结果的好第一步。然而，这还不够，你需要依赖经典验证技术，所以让我们接下来讨论一下。

## 经典验证

你如何验证结果取决于问题领域。在 Web 开发中，你可以使用各种不同的工具和技术，例如：

+   **测试**：通过端到端测试或前端测试，你可以验证代码是否按预期工作。通常，这种测试涉及使用程序化方法来模拟用户与网页的交互，例如使用 Selenium 等。

+   **手动测试**：你可以通过在浏览器中打开网页并与它交互来手动测试网页。这是在验证过程开始时使用的一个好方法。除了交互之外，你还可以根据你的要求从视觉上检查网页是否正确。

+   **代码审查**：你可以审查代码，看看它是否看起来正确。这是在验证过程开始时使用的一个好方法。这不仅允许你验证结果，还允许你的同行进行验证。

+   **工具**：工具可以测试各种不同的场景，如可访问性、性能等。这些工具很可能已经是你的开发过程的一部分了。

在进行数据科学时，你可能依赖于所有上述方法，但你也可以使用其他方法。一些常见的方法包括：

+   **单元测试**：你可以使用单元测试来验证代码是否按预期工作。

+   **集成测试**：你可以使用集成测试来验证代码是否按预期工作。

+   **结果验证**：这种验证方式意味着你将你的分析或模型的结果与已知结果或基准进行比较。

+   **交叉验证**：这种验证方式意味着你将数据分为训练数据和样本数据，在训练数据上训练你的模型，并在样本数据上评估它。这是在验证过程开始时使用的一个好方法。
