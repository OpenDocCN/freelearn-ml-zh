

# 第一章：这是一个新世界，一个拥有人工智能助手的世界，而你被邀请加入。

# 简介

2022 年 11 月，ChatGPT 似乎从天而降。随着时间的推移，ChatGPT 获得了动力，逐渐演变成为一个被广泛接受的工具。最终，数百万用户将 ChatGPT 整合到他们的工作流程中，利用其生成见解、总结文本、编写代码等功能。

它的出现改变了许多人的工作流程，并在快速理解大量文本、撰写电子邮件等任务上有了很大的改进。你现在购买了这本书，希望学习如何使用像**ChatGPT**或**GitHub Copilot**这样的 AI 工具来提高你的效率。这正是本书的使命：不仅教你如何使用这两个 AI 工具，还能让你能够在各个问题领域应用它们。

在我们开始使用 AI 助手解决问题之前，让我们回顾一下；我们是如何到达这里的？ChatGPT 并不是突然出现的，对吧？

# ChatGPT 是如何从 NLP 到 LLMs 发展而来的

为了讲述我们如何到达这里，到达像 ChatGPT 这样的 AI 工具，由**大型语言模型**（**LLMs**）提供动力，让我们首先了解一下**自然语言处理**（**NLP**）。

NLP 是计算机科学、人工智能和计算语言学的一个领域。它关注计算机与人类语言之间的交互，以及如何编程计算机来处理和分析大量自然语言数据。NLP 是一个极其有趣的领域，在现实世界中有着广泛的有用应用。以下是一些例子：

+   **语音识别**：如果你有一部现代智能手机，你很可能已经与 Siri 或 Alexa 等语音助手互动过。

+   **机器翻译**：当想到机器翻译时，可能首先想到的是谷歌翻译，它具有从一种语言自动翻译到另一种语言的能力。

+   **情感分析**：一个非常有用的领域是理解社交媒体等领域的情感。公司想知道品牌是如何被感知的；电子商务希望快速理解产品评论以促进其业务。

+   **聊天机器人和虚拟助手**：在 ChatGPT 出现之前，你可能已经看到聊天机器人被集成到网页上。这些聊天机器人可以回答简单的问题，公司使用它们来确保你能够快速得到简单问题的答案，并提供比 FAQ 页面更自然的使用体验，以及其他用途领域。

+   **文本摘要**：当想到文本摘要时，我们再次想到搜索引擎。你可能见过，当你使用 Bing 或 Google 等搜索引擎时，它能够总结一个页面，并在搜索结果页面上与页面的链接一起显示摘要。作为用户，你能够更好地理解应该点击哪个链接。

+   **内容推荐**：这是被众多不同领域使用的一个重要领域。电子商务用它来展示你可能感兴趣的产品，Xbox 用它来推荐游戏以及购买建议，而视频流媒体服务则显示你可能想观看的内容。

如您所见，通过 NLP，无论是公司还是最终用户，都能从中获得巨大的好处。

## LLMs 的兴起

我们是如何从 NLP（自然语言处理）发展到 LLMs（大型语言模型）的呢？最初，NLP 使用基于规则的系统和统计方法。这种方法虽然对于某些任务效果不错，但在处理人类语言方面遇到了困难。

当深度学习（机器学习的一个子集）被引入 NLP，并产生了如 RNN（循环神经网络）和基于 transformer 的模型等能够学习数据中模式的新模型时，情况发生了改变，性能得到了显著提升。基于 transformer 的模型，我们开始为大型语言模型奠定基础。

LLMs 是一种 transformer 模型。它们可以生成类似人类的文本，并且与 NLP 模型不同，它们擅长各种任务，而无需特定的训练数据。这是怎么做到的呢？答案是改进的架构、计算能力的巨大提升以及庞大的数据集。

LLMs（大型语言模型）基于这样一个理念：一个足够大的神经网络，在足够的数据和计算资源支持下，可以学会做任何事情。这标志着我们在编程计算机方面的范式转变。我们不再编写代码，而是编写提示并让模型完成剩余的工作。

## GPT 模型

目前有各种各样的 LLMs，但让我们先关注一下 GPT，这是一种 LLM，本书所选的工具就是基于它的（即使 GitHub Copilot 使用的是名为 Codex 的特定子集）。

在过去几年中已经开发了几个不同的版本。以下是一些由 OpenAI 公司开发的模型：

+   GPT-1：第一个模型，拥有 1.17 亿个参数，采用了 transformer 架构。

+   GPT-2：这个模型拥有 15 亿个参数，能够生成连贯且相关的文本。

+   GPT-3：这个模型拥有 1750 亿个参数，在回答问题、生成小说甚至编写代码等特性上，比其前身有了显著的提升。

+   GPT-4：据称，这个模型拥有 1.76 万亿个参数。

+   参数的数量使得模型能够理解更加细腻和连贯的文本。还应该指出的是，模型越大，训练它所需的计算资源也就越大。

+   ChatGPT 最近切换到了 GPT-4，与 GPT-3 相比，差异非常显著。

## LLMs 的优势在哪里

现在我们已经更好地理解了 LLMs 是如何产生的以及它们的来源，那么是什么让 LLMs 如此出色呢？有哪些好的例子说明我们真的应该采用基于 LLMs 的人工智能助手呢？

由于 LLMs 更大、更先进，它们在某些方面明显优于传统的 NLP 模型：

+   **背景**：大型语言模型（LLMs）不仅能理解最近的输入，还能基于更长的对话产生回应。

+   **少样本学习**：为了执行任务，LLMs 通常只需要几个示例就能产生正确的响应。这与通常需要大量特定任务训练数据的 NLP 模型形成对比。

+   **性能**：在翻译、问答和摘要等领域，LLMs 比传统的 NLP 模型表现更好。

值得注意的是，LLMs 并不完美；它们确实会生成错误的响应，有时也会编造响应，也称为幻觉。但我们希望，通过阅读这本书，你将看到使用基于 LLM 的人工智能助手的优势，并且你会觉得优点明显大于缺点。

# 新范式：用自然语言编程

使用基于 LLM 的人工智能助手最大的变革可能是，你能够仅使用自然语言与之互动。你不需要学习编程语言来获取你需要的响应。这种变化构成了与 AI 互动的新范式。我们正在从为生成应用程序、数据检索甚至我们生成图像、演示文稿等方式，转向通过提示以高层次表达我们想要的内容，而不是用特定的语言编写。

这里有一个例子，展示了现在使用提示可以实现的事情，而以前这需要相当多的努力：

+   **编程**：通过提示，你表达你想构建的应用程序或你想要用代码做出的更改。

+   **图像生成**：以前你需要设计师或艺术家，现在你可以通过提示生成。

+   **视频**：有一些工具，一旦给出提示，就能生成由头像朗读你书面文本的视频。

+   **文本任务**：基于 LLM 的人工智能助手可以生成电子邮件、总结大量文本、撰写访谈广告等等；任何你可以用文本想象的事情。

所以上述提到的所有应用领域都清楚地表明，基于 LLM 的人工智能工具不仅对程序员和数据科学家有用，对众多不同的职业也是如此。

## 挑战与限制

目前一切是否运行得完美？人工智能助手还无法完全取代“你”，应该被视为更多是一种“思考伙伴”。微软甚至通过有意识的命名，将他们的 AI 助手称为“Copilots”，其中你显然是飞行员，负责设定方向。这些工具可以在几秒钟内生成文本和其他模态，但你需要验证其正确性。通常，你从工具那里得到的第一个响应是需要迭代的内容。好消息是，重新执行指令只需要几秒钟。

关于人工智能助手的要点之一是，你在某个特定主题上的技能越熟练，你就能向它提出越智能的问题，你将能够更好地评估响应的正确性。

# 编程语言的演变

编程在历史上经历了一系列的变化和范式转变：

+   在 19 世纪 40 年代，Ada Lovelace 为机器编写了第一个算法，即分析引擎。Lovelace 被认为是第一位计算机程序员，也是第一位认识到机器的应用不仅限于纯计算的人。

+   在 20 世纪 40 年代，创建了第一批可编程计算机。这些计算机使用穿孔卡片编程。其中之一是哈佛马克 I 号，它被用来计算炮弹的弹道。还值得一提的是 Bombe，它在第二次世界大战期间被用来破解恩尼格玛密码，对盟军赢得战争起到了关键作用。

+   在 20 世纪 50 年代，创建了第一批高级编程语言。这个时期见证了 FORTRAN、LISP、COBOL 和 ALGOL 的诞生。其中一些语言至今仍在使用，尤其是在银行系统、科学计算和防御领域。

+   在 20 世纪 70 年代，创建了第一批面向对象的编程语言。70 年代意味着我们得到了 Smalltalk、C++和 Objective-C。除了 Smalltalk 之外，这些语言至今仍在广泛使用。

+   在 20 世纪 90 年代，创建了第一批函数式编程语言。90 年代为我们带来了 Haskell、OCaml 和 Scala。这些语言的好处是它们鼓励不可变性和纯函数，这使得它们更容易推理和测试。

+   在 2000 年代，创建了第一批声明式编程语言。声明式编程语言用于描述你想做什么，而不是你想如何做。2000 年代为我们带来了 SQL、HTML 和 CSS。

+   在 2010 年代，创建了第一批低代码和无代码平台。这些平台使编程面向更广泛的受众，并允许任何，无论技术背景如何，都能构建应用程序。

+   在 2020 年代，创建了第一个利用自然语言的 AI 助手。如果你能写一句话，你就能写代码。

总结来说，编程经历了一系列的变化和范式转变。提示优先编程是最近的范式转变，掌握它将是保持相关性的关键。

## 展望未来

有理由感到兴奋，因为我们正在比以前更快地前进，但一如既往，我们应该保持谨慎。我们应该意识到使用这些工具不负责任的风险和危险，但最重要的是，我们应该意识到机会。
