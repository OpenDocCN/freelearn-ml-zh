

# 第二十二章：结论

感谢您，亲爱的读者，阅读到这本书的这一部分。我们希望您现在能够自信地在使用 GitHub Copilot 和 ChatGPT 等 AI 工具的项目中。

# 书籍回顾

让我们回顾一下这本书中我们涵盖的内容。我们首先向您介绍了 AI 的世界以及我们如何到达**大型语言模型**（**LLMs**）。然后，我们向您介绍了 GitHub Copilot 和 ChatGPT，这两者是目前世界上最受欢迎的 AI 工具之一。这两个工具不仅受欢迎，而且值得比较和对比。ChatGPT 带有聊天界面，设计用来处理各种任务。GitHub Copilot 也带有聊天界面和编辑器模式，但更专注于解决软件开发中的问题。这两个工具的一个重要共同点是，您可以使用提示和自然语言输入，它们可以作为输入使用，并且最终用户会收到一个输出，希望这能帮助他们更接近解决问题。因为这两个工具都依赖于提示输入，这使我们能够决定需要什么类型的提示和多少提示，调整提示，甚至使用提示来验证 AI 工具的响应。

然后，我们向您展示了一种提示策略，以确保您有效地使用这些 AI 工具。之后，我们展示了这种提示策略如何在跨越前端到后端、涵盖几个章节的 Web 开发项目中得到应用。然后，我们展示了同样的提示策略如何应用于数据科学和机器学习项目。这两个问题领域是随机选择的；这本书的目的是展示生成式 AI 工具的能力，并且知道如何将它们与提示策略相结合将极大地增强您的能力。

使用 AI 工具的影响是，您现在拥有一个工具，它可以处理代码开发中的大部分繁重工作，无论问题领域如何。这对您作为开发者的意义在于，您现在可以更专注于声明性，即表达您想要的事情，而不是逐行编写代码。使用 AI 工具并高效地使用它们可能会显著加快您工作中编码的部分。

在总结这本书之前，我们向您展示了对未来的展望，即代理，这些是代表您行事并代表我们认为 AI 将走向何方的程序。

最后，我们更详细地介绍了 AI 工具及其功能，以便您在未来的项目中取得成功。

# 主要结论

那么，我们能从这本书中得出哪些主要结论呢？它们如下所示：

+   类似于 GitHub Copilot 和 ChatGPT 这样的 AI 工具将长期存在，并且随着时间的推移只会变得更好。

+   世界正在向更声明性的编程方式转变；让 AI 为您做大部分繁重的工作。您的新角色是与 AI 工具迭代合作，以获得最佳结果。

+   无论你是网页开发者、数据科学家还是机器学习工程师，你都应该制定一个提示策略来处理你选择的领域内的问题。

+   提示（Prompting）正迅速成为一项独立技能，即使我们认为改进后的模型将来会使其变得不那么重要。目前，这是一项你应该掌握的技能，我们希望这本书能帮助你在这条成为熟练提示师的道路上。
